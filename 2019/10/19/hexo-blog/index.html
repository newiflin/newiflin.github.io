使用 Hexo 和 GitHub 搭建个人博客

发表于 2019-10-19 更新于 2019-10-20 分类于 博客

前言

很早就计划着写博客的念头，从申请域名，租服务器，又因为各种原因推到现在。最近无意浏览到了搭建基于 Hexo 与 GitHub 的教程，废话少说，Just Do It。

环境

  • node.js（推荐 10.0 版本及以上）
  • git
  • hexo

Node.js

什么是Node.js

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时环境，运行在服务端，具有开源、跨平台的特性。

Node.js 大部分基本模块使用 JavaScript 编写，使用事件驱动、非阻塞和异步输入输出模型等技术提高性能。

Node.js安装

登录官网下载：https://nodejs.org/zh-cn/download/

下载 windows 安装包，安装一路 next 即可。

查看是否安装成功，运行 node -v，出现版本号。

git

什么是git

git是优秀的分布式版本控制系统。

git的安装

  • windows安装，直接访问 git官网下载。根据电脑位数，选择32/64bit 对应安装即可。
  • 打开 Git Bash 设置git用户名和邮箱

GitHub

GitHub 是目前主流的开源代码托管平台，同时也是程序员的社交和学习平台。

  • 注册GitHub账号

  • 创建一个私有仓库（私有仓库已免费，微软于去年收购GitHub）

Hexo

什么是Hexo

Hexo 是一个快速、简单和功能强大的博客框架，支持 markdown 等标记语言，可以立即生成带有优美主题的静态文件。

Hexo官方文档：https://hexo.io/docs/

安装Hexo

  • 安装Hexo之前，确保已经按照 Node.js 和 Git

  • 在H盘新建文件夹blog

  • cmd运行命令npm install -g hexo-cli

    报了两个错误，不过并不影响后续。

  • 运行hexo init newiflin.github.io，创建博客目录

  • 启动hexo服务

  • 启动Chrome，访问http://localhost:4000，预览页面

  • 发表文章，命令hexo new [文章名]

会在source/_posts目录下生成对应的 md 文件，可预览页面。

  • 修改hexo目录下的 _config.yml 文件

    1. 修改 deploy 部分

    2. 修改 url 和 root

    3. 模板默认为 lanscape，可修改配置文件，更换主题
